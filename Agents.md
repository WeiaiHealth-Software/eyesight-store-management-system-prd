# 门店管理系统与小程序预约模块 - 角色说明 (Agents)

本文档梳理了门店管理系统及小程序预约家庭组模块中涉及的所有核心角色（Agents）及其对应的职责、权限与系统交互逻辑。

## 1. 患者 / 家庭组成员 (Patient / Family Member)
**角色描述**：门店的终端服务对象，包括初诊患者、复查患者及进行视觉训练的患者。
**系统模块**：微信小程序 (个人中心、我的家庭组、我的就诊人)
**核心功能与权限**：
- **自助预约**：通过微信小程序一键登录后，选择就诊日期与时间段完成门诊预约或主动预约视觉训练。
- **家庭组管理**：账号绑定家庭ID，支持添加多个家庭成员。解决家长轮流带孩子就诊时手机号不一致的问题。
- **多设备支持**：同一账号可在多台设备登录，通过扫码绑定新设备。
- **消息接收**：预约提交被前台确认后，自动接收预约成功通知；知晓系统因满号而限制预约的实时状态。

## 2. 前台 (Receptionist)
**角色描述**：门店的第一接触点，负责接待、建档以及基础的预约统筹管理。
**系统模块**：前台面板 (Pad端/PC端)
**核心功能与权限**：
- **搞定预约，一气呵成**：处理日常预约审核与排期。
- **预约审核**：收到患者小程序提交的预约弹窗提醒，并确认是否可接诊。
- **代预约与建档**：针对首次到访、操作困难或需快速安排复查的患者，引导其扫码注册，并在CRM用户管理面板帮其快捷预约排期。

## 3. 医生 (Doctor)
**角色描述**：负责常规门诊的诊断与治疗方案制定的医疗专业人员。
**系统模块**：医生面板 (Pad端/PC端)
**核心功能与权限**：
- **专注问诊，信息直达**：主要处理门诊预约挂号的患者。
- **接诊与诊断**：查看患者病历信息，进行常规问诊，并可为患者开具视觉训练等后续治疗方案。

## 4. 视光师 (Optometrist)
**角色描述**：眼健康的全科医生，不仅负责验光配镜，还负责诊断和处理视觉功能异常以及视力康复等。
**系统模块**：视光师面板 / 诊室面板 (Pad端)
**核心功能与权限**：
- **跟着人走，动态占位**：进入诊室后，在固定于诊室的Pad上进行快速登录（如头像点击+四位密码）以占用诊室和设备。
- **方案制定与协同**：负责诊断并制定视觉训练方案，与视训师沟通具体的训练预约逻辑。
- **任务调拨**：因可能跨检查室作业，系统需支持其任务调拨和全流程数据采集功能。

## 5. 视训师 (Vision Therapist)
**角色描述**：视觉训练的教练，更像是患者的视觉教练，在视光师或医生的诊断和方案指导下，具体执行视觉功能训练。
**系统模块**：Pad端预约模块
**核心功能与权限**：
- **执行训练**：为具有较高粘性、已有治疗方案的患者执行每周固定时间或每月固定次数的视觉恢复训练。
- **固定预约管理**：通常由视训师通过平板，直接帮助患者预约固定时间的视觉训练（不挂医生号）。

## 6. 系统管理员 / 后台管理 (System Administrator)
**角色描述**：负责门店系统规则配置、号源统筹与紧急情况处理的后台人员。
**系统模块**：后台管理系统
**核心功能与权限**：
- **号源与规则配置**：设置每半小时时间段的容纳人数，合理配置初诊与复查患者的时间占用比例。
- **特需号管理**：设置普通号（如41元）与特需号（如300元）的数量与规则，进行客户分层服务。
- **紧急加号权限**：在系统防超约机制锁定可用号源、显示满员的情况下，保留后台加号的最高权限，以应对紧急突发情况。
